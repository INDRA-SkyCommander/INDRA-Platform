# Modules (research prototypes)

## What's in this folder

### Packet / command manipulation
- **`cmdinjector.py`** - Prototype that inspects UDP control traffic and can rewrite/forward payloads while capturing local keyboard input.

### SDR / GNU Radio signal generators
- **`mpsk_stage1.py`** - GNU Radio flowgraph (Python) using `osmosdr` to transmit a synthesized complex signal. (Generated by GRC.)
- **`dumbjammer.py`** - Similar GNU Radio flowgraph script intended as a minimal template for SDR signal generation.
- **`cyclic_jammer.py`** - Variant that programmatically changes the SDR center frequency over time.
- **`Revised_Jammer.grc`** - GNU Radio Companion (.grc) source for one of the SDR flowgraphs (open/edit in GRC).

### YAML-driven SDR transmitter wrapper
- **`jammer.py`** - Reads `jaml.yaml` and configures an `osmosdr` sink with an analog noise source in GNU Radio.
- **`jaml.yaml`** - Configuration values consumed by `jammer.py` (frequency, sample rate, bandwidth, duration, etc.).

### Wi-Fi frame generation / capture helpers
- **`rf_jammer.py`** - Generates randomized 802.11 data frames and transmits them through a selected interface; reads target/options from `data/module_input_data.json`.
- **`video_sniffer.py`** - Captures 802.11 frames from a specific source MAC and reconstructs base64-encoded H.264 NAL units into `data/sniff_output.log`.
- **`video_interceptor.py`** - Orchestrates an external sniffer/decoder pipeline and manages cleanup and output directories.

## Expected project files / outputs

Some modules assume these repo paths exist:

- `data/module_input_data.json` - input describing the target and selected options (interface, packet counts, etc.).
- `data/sniff_output.log` - log/stream output used by the video tools.
- `data/images/` - directory used for decoded frames/images by the interceptor pipeline.

## Dependencies (high level)

Depending on the module, you may need:

- Python 3.x
- `scapy` (packet parsing/crafting)
- GNU Radio 3.10 + `osmosdr` (SDR sink/source blocks)
- `PyYAML` (YAML config loading)
- OS-level wireless/SDR drivers + appropriate hardware

## Legacy GUI (deprecated)

These files are kept from an older Tkinter-based GUI build. They are **not required** for the current CLI/module usage,
and may be incomplete or out of date.

- **`GUI.py`** - Tkinter application entry point for the older INDRA GUI (host list, scan controls, module selection, log window, image player).
  It coordinates scans and module launches and writes selected state to `data/module_input_data.json`.
- **`colors.py`** - Shared color constants used by the Tkinter GUI theme (hex color strings).
- **`skyjackold.py`** - Legacy, unfinished proof-of-concept script for interacting with nearby drone Wi-Fi access points (contains stubbed logic and rough parsing).
  Retained for historical reference only.
